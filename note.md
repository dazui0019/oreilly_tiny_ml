## 线性回归

注意: 写的比较混乱, 然后我这里"误差"和"损失"混用了。

![线性回归](images\Okuns_law_quarterly_differences.png)
有些数据可能会满足线性分布, 比如ADC的采样值与温度值或者电压值理论上应该满足一个线性函数$y=k*x+b$, 但是由于采集误差等原因我们采集到的ADC值会离散的分布在这条线性函数的周围。并且由于传感器在不同电压/温度的情况下误差也会有所不同, 比如NTC在$10^\circ\mathrm{C}$和$100^\circ\mathrm{C}$下的误差肯定是不同的, 因此我们要找到一个误差最小的模型(**找到一条误差最小的直线**)。

### 损失函数

实际值与我们模型之间的差值, 我们称为损失。

我们把"实际数据"到"模型预测值"的均方差(因为两数之差会存在负数, 所以要平方一下)。我们最终就是要确定模型 $y=a*x+b$ 中某个 $k$ 和 $b$ 的组合, 使得我们模型的误差最小。
$$Loss = \frac{1}{n} \sum_{i=1}^{n} (y_{预测}^{(i)} - y_{真实}^{(i)})^2$$
![误差](images\误差.png)

因为不同的 $k$ 和 $b$ 的组合(不同的直线), 都会得到一个误差值。但是这样的 $k$ 和 $b$ 的组合会有无数多个, 人类无法穷举出所有的情况。

| 直线编号 | 斜率($k$) | 截距($b$) | 误差($z$) |
| :---: | :---: | :---: | :---: |
| $1$ | $k_1$ | $b_1$ | $z_1$ |
| $2$ | $k_2$ | $b_2$ | $z_2$ |
| ... | ... | ... | ... |
| $n$ | $k_n$ | $b_n$ | $z_n$ |

但是误差 $z$ 是随着斜率 $k$ 和截距 $b$ 的变化而变化的, 所以误差值就是一个以直线斜率和截距为自变量的二元函数 $z = \phi(k, b)$ , 形状类似图片这样。我们要找的 $z$ 最小的那个点, 就是函数图形上最底部的点。

![](images\二元函数.png)

### 找到损失最小的点

我们把模型函数代入刚才的均方差公式中, 并且 $[x^{(i)}, y_{真实}^{(i)}]$ 是在我们数据集里面的已知数据。于是我们得了到一个以直线斜率和截距为自变量的二元函数 $z(k, b)$ 。

$$z(k, b) = \frac{1}{n} \sum_{i=1}^{n} (k \cdot x^{(i)} + b - y_{真实}^{(i)})^2$$

这个函数的意义就是不同的模型(不同的 $k$ 和 $b$ ), 与每一个散点的距离的均方差 (在机器学习里叫做**损失**, 因为一般一个点上的差值才会叫误差吧, 那多个点的平均误差, 就叫做模型的损失) , 总之该函数可以找到不同模型与实际散点 (在不同 $x$ 上采集到的真实数据 $y$ ) 均方差的所有可能性。

我们就是要找到一个 $k$ 与 $b$ 的组合, 使得损失 $z$ 最小, 也就是三维平面上谷底的点。为了找到这个点, 我们可以有两种解法, 其中一个最容易想到的, 就是分别找不同维度上的极小值, 这个叫做**坐标下降法**。

![](/images/偏导的意义.png)

#### 坐标下降法

首先随便固定一个维度 $k_1$ 将它代入我们的损失函数, 找到单一方向上的极小值, 解算到的 $b_1$ 代入损失函数计算另一个方向上的极小值, 解算得到 $k_2$, 再将 $k_2$ 重新代入损失函数计算。

这个方法很简单, 对于上图这样的函数, 我们只需要计算两次就能得到最小值。但是实际的损失函数来说会非常的复杂, 所以没有办法直接求出曲线上的最小值, 所以一般都是通过求导的方式来判断该点是否在曲线上的低谷, 如果在某一点上的导数为0, 则就说明该点处于一个谷底。

然后导数其实是包含方向坐标的, 所以我们按照导数方向相反的方向走就能找到谷底, 我们会根据导数的大小来移动一个距离。比如我们预设了一个 $b_1$, 那么我们下一个预测的 $b_2$ 就等于 $b_1 - \alpha\times\frac{\Delta{z_1}}{\Delta{b_1}}$, 其中 $\alpha$ 是学习率, $\frac{\Delta{z_1}}{\Delta{b_1}}$ 是 $b_1$ 这点的切线斜率。学习率是一个可以动态调整的值, 是我们自己选定的一个常量, 当然取值也有讲究。

然后就通过这种方式分别找到各条曲线上的最小值, 然后反复计算几次应该就能找到函数的最小值。

#### 梯度下降法

上面我们是讨论单个维度上的斜率, 如果有多个维度的话, 可能就需要计算很多次。而梯度下降算法则是将多个维度上的偏导(就是不同维度上的斜率)组合成一个多维的向量。这个向量就叫做梯度。
$$\nabla J(w, b) = \begin{bmatrix} \frac{\partial J}{\partial w} \\ \frac{\partial J}{\partial b} \end{bmatrix}$$
相比于单一维度上的斜率, **梯度坐标会更直接的指向整个多维平面上的最小值**, 而一维的斜率可能会因为不同维度互相牵制而产生zig-zag的现象。例如下面这个函数, 由于存在一个交叉项 $2xy$ 所以每一次对 $x$ 或者 $y$ 的调整都会整体影响 $2xy$。

这说明两个方向不是独立的。对坐标下降来说，这会带来两个现象：

1. 每次沿一个坐标方向优化时，结果都强烈依赖另一个坐标当前的位置。
2. 更新 x 后，会改变接下来 y 的最佳更新方向，反过来也一样

所以算法会出现“来回拉扯”：

- 先固定 y 优化 x
- 再固定 x 优化 y
- 但每次更新都会改变对方的最优点

![zig_zag](/images/zig_zag.png)

梯度下降的具体的细节先不深究了。

### 创建模型

使用 `TensorFlow` 的 `Keras` 工具。

``` python
# 创建一个空的序列模型框架
# 后面add()的神经元层会依次添加到该框架上
model_1 = tf.keras.Sequential()

# 往模型框架里面添加神经元层
# Dense表示全连接层, 前一级的输入会连接到Dense层的每一个神经元
# input_shape=(1,), 表示输入数据的维度, 这里是一个1维向量
model_1.add(keras.layers.Dense(8, activation='relu', input_shape=(1,)))
# 也是一个Dense层而且只有一个神经元, 前一级的8个神经元会全部连接到该神经元上
model_1.add(keras.layers.Dense(1))


model_1.compile(optimizer='adam', loss='mse', metrics=['mae'])
model_1.summary()
```

#### 激活函数

机器学习（尤其是监督学习）的本质确实可以看作是在高维空间中进行函数拟合。而对于现实生活中出现的情况, 肯定无法使用一个线性函数来拟合, 因此需要一个非线性的激活函数来让训练后的模型非线性化。

下面这张图是不加激活函数的情况, 每一个神经元的输出函数就只是 `input` 与 `weight` 和 `bias` 的线性关系 $y = wight*x+bias$。假如是一个两层神经元的结构, 那么将上一次神经元的输出函数作为下一个神经元的输入可以得到 $y = wight*(wight*x+bias)+bias$, 系数wight相乘还是一个常量, 所以这任然是一个线性函数。因此如果不引入激活函数, 那么即使增加再多层的神经元, 最终也只是一个线性函数。

![无激活函数](images\无激活函数.png)

但是激活函数也分好坏, 下图引入就是一个不好的激活函数 $\phi(x)=x^2$, 引入该函数会导致输入`x`传导至末端时变得非常大, 甚至让计算机无法计算, 这个叫 **数值爆炸**。
